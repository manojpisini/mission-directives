## Usage rules

- Use `office-hours` or `plan-ceo-review` before committing to strategy, organization, or product direction.
- Use `plan-eng-review`, `spec`, and `autoplan` before bounded engineering execution.
- Use `qa`, `review`, `careful`, `guard`, and `design-review` as independent checks, not self-approval.
- Use `investigate` and `debug` for evidence gathering; use matching suite executive prompts for authorized fixes.
- Use `ship`, `land-and-deploy`, and release skills only under `APPLY_APPROVED`.
- Use browser and scraping skills only for authorized public retrieval with provenance, rate, privacy, and injection controls.
- Use design skills for interfaces, diagrams, documents, and layouts; image generation remains excluded.
- Because repository-level audit signals were mixed during research, inspect each exact skill before installation.
